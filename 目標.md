# ♟️ 五子棋對戰平台 - 專案目標檔案 (Goals Document)

## 🎯 總體目標 (Overall Goal)

開發一個基於 **React / Next.js** 的高性能五子棋對戰網頁應用程式，具備完整的**註冊/登入**、**積分系統**、**即時對戰**功能，並遵循現代軟體工程的**可維護性**和**清晰架構設計**標準。

## 🖥️ 核心功能需求 (Core Functional Requirements)

### 1. 介面與導航 (UI & Navigation)

* **單一主頁大廳 (Lobby as Homepage):**
    * 使用者進入網站後直接顯示大廳介面。
    * **大廳列表 (Left Panel):** 顯示當前已建立的私人房間或公開房間列表，使用者可選擇加入。
    * **開始匹配區 (Right Panel):** 包含「**隨機匹配**」和「**建立房間**」的按鈕或選項。
* **遊戲對戰介面 (Game Board):** 顯示五子棋盤、雙方資訊、計時器、以及互動按鈕。

### 2. 使用者與帳號系統 (User & Account System)

* **註冊/登入：** 支援標準的帳號綁定機制。
* **積分制度：** 必須包含積分系統（如 ELO/Glicko-2）。
* **對戰紀錄 (History):** 帳號必須保留完整的對戰歷史紀錄。
* **段位系統：** 根據積分劃分，段位需記錄在帳號中，包括：
    * 銅 (Bronze)
    * 銀 (Silver)
    * 金 (Gold)
    * 白金 (Platinum)
    * 鑽石 (Diamond)
    * 大師 (Master)
    * 頂級 (Apex / Grandmaster)

### 3. 遊戲邏輯與互動 (Game Logic & Interaction)

* **悔棋功能：** 支援雙方同意或單方請求（需定義規則）的悔棋機制。
* **對戰結束後再來一局：** 支援在對局結束後，雙方可選擇快速發起「再來一局」的請求。
* **適當的對手互動：**
    * **即時聊天功能：** 支援對戰中的簡單文本交流。
    * **表情符號/快捷短語：** 支援非語言的即時互動。
* **匹配機制：** 支援**隨機匹配**（基於積分或段位）和**房間加入**兩種模式。

## ⚙️ 技術與部署要求 (Technical & Deployment Requirements)

### 1. 核心技術棧

* **前端：** React + Next.js
* **後端：** 需包含一個持久運行的 **WebSocket Server** 以實現即時通訊。
* **資料庫：** PostgreSQL (透過 Supabase 管理)。

### 2. 部署環境 (DevOps)

* **前端/API：** Vercel
* **即時後端：** Render
* **資料庫：** Supabase
* **工作流程要求：** 在部署時，**必須**優先考慮和使用 **Vercel CLI** 和 **Render CLI** 進行自動化和指令行操作，以優化**編碼工作流程**。
